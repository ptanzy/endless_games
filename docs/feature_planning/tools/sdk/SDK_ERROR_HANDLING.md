# SDK_ERROR_HANDLING

---

## 🎯 Purpose
Define how SDK tools detect, classify, and report errors to creators.

---

# 1. Error Categories

### Validation Errors
- Unsafe logic  
- Invalid assets  
- Performance violations  

### Build Errors
- Packaging failures  
- Missing dependencies  
- Version conflicts  

### Runtime Errors
- Script exceptions  
- Missing references  
- Invalid interactions  

### Publishing Errors
- Metadata issues  
- Permission issues  
- Review rejections  

---

# 2. Error Reporting
- Clear error messages  
- Line/asset references  
- Suggested fixes  
- Severity levels  
- Links to documentation  

---

# 3. Error Handling Rules
- Errors must be deterministic  
- Errors must be reproducible  
- Errors must be logged  
- Errors must be explainable  

---

# 4. Anti‑Patterns
- Silent failures  
- Cryptic error messages  
- Non‑actionable errors  

---
