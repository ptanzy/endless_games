# PLANNING_OVERVIEW

---

## 🎯 Purpose
Provide a unified map of the EndlessGames planning pillar and how all planning documents, folders, and systems relate.

---

## 🧱 Planning Structure

### Platform‑wide governance docs (this folder)
- PLATFORM_DOCTRINE_OVERVIEW  
- CROSS_PILLAR_INTEGRATION  
- TERMINOLOGY_AND_DEFINITIONS  
- PLATFORM_DEPENDENCY_MAP  
- PLATFORM_OWNERSHIP_MATRIX  
- SYSTEM_LIFECYCLE_MANAGEMENT  

### Domain folders
- `/brand` — brand partnerships, ambassadors, swag, brand media
- `/core` — identity, safety, progression, ranks, achievements
- `/cosmetics` — cosmetic system, layers, rules, pipelines
- `/creator` — creator tools, monetization, SDK, creator media
- `/economy` — currencies, ads, affiliate, token rules, membership
- `/future` — far‑future and speculative systems
- `/media` — Tube, TV, Stream, Radio, Moments, Recaps
- `/social` — profiles, clans, preset communication, identity surfaces
- `/tools` — observability, performance, release, testing, internal tools

---

## 📐 Standard Planning Doc Shape

Each planning doc should generally include:

1. Purpose  
2. Goals  
3. Personas  
4. User Stories  
5. Core Capabilities  
6. Workflows  
7. Rules & Constraints  
8. Deep Dive  
9. Interactions  
10. Risks  
11. Success Criteria  
12. Future Extensions  

---

## 🧭 How to Navigate

- Start here, then read `PLATFORM_DOCTRINE_OVERVIEW`
- Use `TERMINOLOGY_AND_DEFINITIONS` to align naming
- Use `CROSS_PILLAR_INTEGRATION` and `PLATFORM_DEPENDENCY_MAP` to understand relationships
- Use `PLATFORM_OWNERSHIP_MATRIX` to know who owns what
- Use `SYSTEM_LIFECYCLE_MANAGEMENT` when adding, changing, or deprecating systems
