# Parental Dashboard Planning

This document defines the parental dashboard features, controls, and reporting for guardians.

---

## 🎯 Purpose
Define the future parental dashboard that allows parents to view their child’s activity, manage playtime, and understand platform safety — without exposing any unsafe communication or personal data.

---

# 1. Goals
- Provide transparency for parents  
- Maintain zero‑moderation, zero‑text safety  
- Support playtime and spending controls  
- Reinforce trust in EndlessGames  

---

# 2. Personas
- **Parent:** Wants visibility and control  
- **Player (Child):** Wants safe, fun gameplay  
- **Platform:** Needs to maintain privacy and safety  

---

# 3. User Stories
- *As a parent, I want to see my child’s playtime.*  
- *As a parent, I want to set limits on spending.*  
- *As a parent, I want to understand platform safety.*  
- *As the platform, I want to protect player privacy.*  

---

# 4. Core Capabilities
- Playtime overview  
- Spending overview  
- EndlessPass progress  
- Game activity summary  
- Safety explanation module  
- Optional playtime limits  
- Optional spending limits  

---

# 5. Workflows

### **5.1 Parent Setup Workflow**
1. Parent creates parent account  
2. Child account linked via code  
3. Parent dashboard unlocked  
4. Parent can view activity and set limits  

---

# 6. Rules & Constraints
- No personal messages displayed  
- No social interactions shown  
- No media uploads shown  
- No identifying information exposed  
- Only high‑level activity summaries allowed  

---

# 7. Deep Dive

### **7.1 Dashboard Sections**
- Playtime Summary  
- Game Activity  
- EndlessPass Progress  
- Cosmetic Purchases  
- Safety Overview  
- Optional Controls  

### **7.2 Optional Controls**
- Daily playtime limit  
- Weekly spending limit  
- Game‑specific restrictions  

---

# 8. Success Criteria
- Parents feel informed and confident  
- Children remain safe and unexposed  
- Platform maintains zero‑moderation safety  
- No personal data is ever revealed
