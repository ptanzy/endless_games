# Copilot Component Generation Rules

## Purpose
Defines how Copilot should generate components for EndlessOS.

---

# Component Rules

## 1. Components Must Be Deterministic
- No random behavior  
- No adaptive layout  
- No dynamic spacing  

## 2. Components Must Be Stateless When Possible
- Prefer pure components  
- Use state only when required  

## 3. Components Must Follow UI Doctrine
- Neutral palette  
- Safe zones  
- Motion rules  
- Identity rules  

## 4. Components Must Be Composable
- No monolithic components  
- No deeply nested logic  

---

# Summary
Copilot must generate components that are deterministic, composable, and aligned with EndlessOS UI doctrine.
