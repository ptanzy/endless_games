# EndlessOS — Typography System

## Purpose
The EndlessOS typography system defines how type is structured, scaled, and applied across the platform.  
It establishes a cinematic, gaming‑grade hierarchy that prioritizes clarity, readability, and long‑term maintainability.

This document outlines:
- Official typefaces  
- Hierarchy and scale  
- Weight usage  
- Line‑height rules  
- Platform‑level display overrides  
- Behavioral constraints  
- Anti‑patterns  

Component‑level typography will be defined in a future specification.

---

# Core Principles

## 1. Content First
Typography must always support the content, not compete with it.  
EndlessOS prioritizes:
- Readability  
- Strong hierarchy  
- Minimal noise  
- Predictable structure  

Type is a functional tool, not a decorative surface.

---

## 2. Cinematic, Gaming‑Grade Clarity
EndlessOS is a dark‑mode‑first platform.  
Typography must remain:
- Crisp  
- High‑contrast  
- Legible at distance  
- Comfortable for long‑form reading  

This aligns with modern console UI standards (PS5, Xbox, Steam, Battle.net).

---

## 3. Deterministic Hierarchy
Typography must behave predictably:
- No dynamic resizing  
- No adaptive scaling  
- No per‑component font overrides  
- No decorative treatments  

Hierarchy is fixed and consistent across the OS.

---

## 4. Identity Through Restraint
Typography is not an identity surface.  
Identity is expressed through:
- Accent colors  
- Motion  
- Layout  
- Platform backgrounds  

Type remains neutral and system‑level.

---

# Typefaces

## Primary Typeface — **Inter**
Inter is the core EndlessOS typeface and is used for **all functional UI**.

### Used for:
- Body text  
- Navigation  
- Labels  
- Buttons  
- System dialogs  
- Cards  
- Lists  
- Metadata  
- Search  
- Settings  
- Dense UI  

### Allowed weights:
- **400** — Body, paragraphs  
- **500** — Labels, secondary emphasis  
- **600** — Head
