# documentation best practices

## Overview
Maintain clear, up-to-date documentation to support development and onboarding.

## Best Practices
- Document code, APIs, and system architecture.
- Use consistent formatting and structure.
- Keep documentation close to the codebase when possible.
- Update docs as part of every code change.
- Use diagrams and examples for clarity.
- Review and prune outdated documentation regularly.
- Encourage contributions and feedback on docs.

# EndlessOS — Documentation Best Practices

## Purpose
Defines how all engineering documentation must be written.

---

# Core Principles

## 1. Clarity Over Completeness
- Write what matters  
- Avoid noise  

---

## 2. Consistent Structure
Each doc must include:
- Purpose  
- Core principles  
- Rules  
- Summary  

---

## 3. Deterministic Language
- No ambiguity  
- No optional behavior  
- No adaptive rules  

---

# Summary
Documentation must be clear, structured, and aligned with EndlessOS doctrine.
