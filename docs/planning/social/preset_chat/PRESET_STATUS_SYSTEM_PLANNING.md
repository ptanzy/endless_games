# PRESET_STATUS_SYSTEM_PLANNING

---

## 🎯 Purpose
Define the preset status system that allows players to express presence, mood, and activity safely.

---

# 1. Goals
- Provide expressive status messages  
- Maintain zero‑moderation safety  
- Support identity and social presence  
- Integrate with games and media  

---

# 2. Personas
- **Player:** Wants to express what they’re doing  
- **Clan Member:** Wants to see clan activity  
- **Creator:** Wants status visibility  

---

# 3. User Stories
- *As a player, I want to set a safe status.*  
- *As a clan member, I want to see who is active.*  
- *As a creator, I want to show when I’m recording or streaming.*  

---

# 4. Core Capabilities
- Preset status library  
- Activity‑based statuses  
- Seasonal statuses  
- Cosmetic status frames  

---

# 5. Workflows

### **5.1 Status Workflow**
1. Player selects status  
2. Status appears on profile  
3. Status appears in clan list  
4. Status appears in EndlessHub  

---

# 6. Rules & Constraints
- No custom text  
- No personal data  
- All statuses must be reviewed  
- Statuses cannot contain unsafe meaning  

---

# 7. Deep Dive

### **7.1 Status Categories**
- Gameplay  
- Social  
- Creator  
- Seasonal  
- Mood (preset‑safe)  

### **7.2 Integration**
- Profiles  
- Clan lists  
- EndlessHub  
- Media surfaces  

---

# 8. Success Criteria
- High status usage  
- Strong social presence  
- Zero moderation incidents  
- Statuses feel expressive
