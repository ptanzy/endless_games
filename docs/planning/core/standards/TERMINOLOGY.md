# TERMINOLOGY

---

## 🎯 Purpose
Define consistent terminology across all EndlessGames planning documents.

---

# 1. Core Definitions

**Platform** — The entire EndlessGames ecosystem.  
**System** — A bounded functional unit with clear inputs/outputs.  
**Feature** — A user‑facing capability within a system.  
**Experience** — Any playable or interactive instance (game, mode, SDK content).  
**Game** — A standalone ruleset built by EndlessGames or via SDK.  
**Pipeline** — A multi‑step process with validation and outputs.

---

# 2. Time‑Based Content

**Event** — Time‑bound platform‑wide content.  
**Series** — Recurring structured content.  
**Quest** — Task‑based objective.  
**Season** — A platform‑wide thematic cycle.

---

# 3. Progression Terms

**Progression** — Long‑term advancement.  
**Achievement** — Milestone earned by players.  
**Reward** — Output granted by a system.  
**Pass** — Seasonal progression track.

---

# 4. EndlessGames‑Specific Terms

**Preset** — A safe, pre‑approved communication unit.  
**Identity Surface** — Any cosmetic expression layer.  
**Hub Module** — A content block in EndlessHub.  
**Creator Tier** — Level of creator capability (Media → Interactive → Game → Mode).  
**Seasonal Arc** — Narrative + content structure for a season.  
**Clan Base** — Cosmetic shared clan space.

---

# 5. Enforcement Rule
All planning documents must use these exact definitions.
