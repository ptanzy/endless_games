# CHESS LOADING SCREEN — UI EXTENSIONS

## Purpose
This document defines chess-specific loading screen enhancements that extend the Generic Loading Screen. These include chess tips, topology previews, and rule reminders.

---

## Extension Areas

### 1. Chess Background Art
Chess may override the generic background with:
- Board art
- 3D topology renders
- Piece silhouettes
- Mode-specific artwork (Cube, Matrix, etc.)

---

### 2. Chess Loading Steps
Chess adds custom loading steps:
- “Initializing Board Topology”
- “Generating Layer Structure”
- “Applying Rule Modules”
- “Placing Player Armies”
- “Calculating Initial Threat Maps”

---

### 3. Chess Tip Bank
Examples:
- “Knights can jump over pieces.”
- “In 4-player chess, checkmate does not end the game.”
- “Vertical movement is only allowed on aligned squares.”
- “Cube mode wraps movement across faces.”
- “Matrix mode uses full 3D vectors.”

---

### 4. Chess Rule Reminder Panel
Displays:
- Mode name
- Movement mode
- Promotion rules
- Elimination rules
- Terrain/portal presence
- Endgame mutation triggers

---

### 5. Chess Player Indicators
- Seat positions (N, E, S, W)
- Team colors
- AI indicators

---

## UX Goals
- Reinforce rules before gameplay
- Provide clarity for 3D modes
- Build anticipation with thematic visuals
