REWARD SYSTEM SPEC (V3 — FINALIZED)
1. Purpose

The Reward System defines:

how value enters the platform
how it is converted into currencies
how currencies are distributed
how exploitation is prevented

This system ensures:

All value is externally grounded, internally controlled, and safely distributed

2. Core Model
2.1 External Value Source

All value originates from:

ad impressions
offerwall purchases

→ No internal value creation

2.2 Multi-Tier Currency System

The platform uses three currencies, each with distinct purpose and control level.

3. Currency Definitions
3.1 EndlessCoin (Base Currency)

Type: Earned
Source: Ad-generated value
Availability: High
Value: Low

Characteristics
earned from:
content views
gameplay activity
engagement
widely distributed
used for:
general participation rewards
low-value incentives
Design Intent

High-volume, low-risk reward layer

3.2 EndlessGold (Premium Currency)

Type: Purchasable + Limited Earned
Source:

direct purchase (primary)
small amounts from:
ad views
offerwall conversions

Availability: Low
Value: High

Characteristics
scarce
higher utility than EndlessCoin
bridges:
real money → platform economy
Design Intent

Controlled premium value layer tied to real-world spend

3.3 Cash-Out Currency (Tier 3)

Type: Earned → Redeemable
Source: System-controlled conversion from value-generating actions

Availability: Extremely Low
Value: Highest

Characteristics
convertible to:
gift cards
tightly controlled
minimal emission
Design Intent

Real-world value extraction layer with strict safeguards

4. Reward Generation
4.1 Core Trigger
Ad Viewed OR Offerwall Completed →
  Value Generated →
    Currency Allocation
4.2 Allocation Model

Each value event distributes across tiers:

Generated Value →
  EndlessCoin (majority)
  EndlessGold (small portion)
  Cash-Out Currency (minimal portion)

⚠️ Constraint:

Allocation ratios are:
system-defined
not user-controlled
5. Reward Distribution Logic
5.1 Distribution Targets

Rewards are allocated to:

players (activity participation)
creators (value generation via content/games)
system (retained portion)
5.2 Creator Advantage

Creators receive:

higher share of value generated by:
their content
their games
6. Anti-Exploitation Model (CRITICAL)
6.1 Risk-Based Reward Caps (CONFIRMED DESIGN)

This is one of your strongest decisions.

Rule:
If activity is EASY to automate → LOW CAP
If activity is HARD to automate → HIGH or NO CAP
Examples
Activity Type	Risk	Cap
Passive content viewing	High	Low cap
Repeated simple actions	High	Low cap
Complex gameplay sessions	Low	High or no cap
Social interactions (friend-based)	Medium	Medium cap
6.2 Enforcement Mechanisms
6.2.1 Rate Limiting
max rewards per:
time window
activity type
6.2.2 Diminishing Returns
Repeated identical behavior →
  rapidly decreasing reward
6.2.3 Interaction Validation
only valid interactions count:
friend-based
non-repetitive
6.2.4 Automation Detection Integration

From Permission System:

if automation detected:
rewards reduced
caps lowered
actions throttled
7. Currency Separation Rules
7.1 No Direct Conversion (Default)
EndlessCoin ≠ EndlessGold ≠ Cash-Out

[ OPTIONAL FUTURE CONTROL ]

9. Constraints
9.1 No Infinite Emission
all rewards tied to:
real value generation
9.2 No Self-Reward Loops

9.3 No Direct User Control

influence reward formulas
set values

EndlessGold Balance
