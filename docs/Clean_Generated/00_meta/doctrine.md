# EndlessGames Doctrine

## 1. Purpose

EndlessGames is a real-time, browser-based gaming platform where:

- players generate value through gameplay
- systems convert that value into rewards
- creators build within constrained, safe systems

The doctrine ensures:

> The platform remains fair, exploit-resistant, scalable, and economically viable.

---

## 2. Core Doctrines

### 2.1 Effort-Based Value

- All rewards must originate from:
  - measurable player effort
- Passive or automated value generation is prohibited

---

### 2.2 Anti-Exploit by Design

- Systems must:
  - prevent exploitation structurally (not just detect it)
- No system should rely solely on moderation

---

### 2.3 Event-Driven Truth

- All systems operate on:
  - immutable event streams
- No system may rely on unverifiable state

---

### 2.4 Constrained Creativity

- Creators build using:
  - predefined templates and rules
- Arbitrary logic is not allowed

---

### 2.5 Safe Social Interaction

- All player interaction must:
  - be constrained
  - be auditable
- Free-form communication is not permitted

---

### 2.6 Economic Sustainability

- Rewards must:
  - scale with real revenue
- No inflationary or unbounded reward systems

---

### 2.7 Zero Moderation Dependence

- Systems must:
  - prevent unsafe behavior automatically
- Human moderation is not a primary control layer

---

## 3. Non-Negotiable Constraints

- No infinite reward loops
- No unbounded scoring systems
- No direct user-to-user value transfer
- No unverifiable inputs into reward systems
- No removal of historical events

---

## 4. System Alignment Requirement

All systems must:

- integrate with:
  - Event System
  - Risk System
  - Configuration System
- respect:
  - identity rules
  - safety constraints
  - economic limits
